Define the ones we actually need. @tags div p em strong ul li div( p("This ", em("is"), " a ", strong("paragraph."), p("And this is ", strong("another"), " one"), ul( li("and"), li("an"), li("unordered"), li("list") ) ) ``` *Notes* All the arguments passed to a node are flattened into a single vector rather than preserving any nested structure. This means that passing two vectors of nodes to a `div` will result in a `div` node with a single vector of children (the concatenation of the two vectors) rather than two vector children. The only arguments that are not flattened are nested nodes. String arguments are automatically converted into text nodes. Text nodes do not have any children or attributes and when displayed the string is escaped using [`escapehtml`](dom.md#Documenter.Utilities.DOM.escapehtml-Tuple{AbstractString}). # Attributes As well as plain nodes shown in the previous example, nodes can have attributes added to them using the following syntax. ```julia div[".my-class"]( img[:src => "foo.jpg"], input["#my-id", :disabled] ) ``` In the above example we add a `class = "my-class"` attribute to the `div` node, a `src = "foo.jpg"` to the `img`, and `id = "my-id" disabled` attributes to the `input` node. The following syntax is supported within `[...]`: ```julia tag["#id"] tag[".class"] tag[".class#id"] tag[:disabled] tag[:src => "foo.jpg"] # ... or any combination of the above arguments. ``` # Internal Representation The [`@tags`](dom.md#Documenter.Utilities.DOM.@tags-Tuple) macro defines named [`Tag`](dom.md#Documenter.Utilities.DOM.Tag) objects as follows ```julia @tags div p em strong ``` expands to ```julia const div, p, em, strong = Tag(:div), Tag(:p), Tag(:em), Tag(:strong) ``` These [`Tag`](dom.md#Documenter.Utilities.DOM.Tag) objects are lightweight representations of empty HTML elements without any attributes and cannot be used to represent a complete document. To create an actual tree of HTML elements that can be rendered we need to add some attributes and/or child elements using `getindex` or `call` syntax. Applying either to a [`Tag`](dom.md#Documenter.Utilities.DOM.Tag) object will construct a new [`Node`](dom.md#Documenter.Utilities.DOM.Node) object. ```julia tag(...) # No attributes. tag[...] # No children. tag[...](...) # Has both attributes and children. ``` All three of the above syntaxes return a new [`Node`](dom.md#Documenter.Utilities.DOM.Node) object. Printing of `Node` objects is defined using the standard Julia display functions, so only needs a call to `print` to print out a valid HTML document with all nessesary text escaped. ,
